# CNN-DQN Pacman Agent Design — Submission 1.2

## Summary

Build a new Pacman agent in `submissions/1.2/` that uses a small CNN-DQN model as its primary decision-maker, falling back to A* pathfinding when the DQN has low confidence or the enemy is hidden by fog of war. The GhostAgent uses the proven rule-based logic from submission 1. All code, weights, and documentation live in a flat folder structure.

## Folder Structure

```
submissions/1.2/
├── agent.py              # PacmanAgent (DQN + A*) + GhostAgent (rule-based from submission 1)
├── model.py              # PacmanCNN model definition (PyTorch)
├── train.py              # Training script: self-play DQN training
├── pacman_dqn.pt         # Trained weights (generated by train.py)

```

No subdirectories. The GhostAgent's helper functions (BFS, dead-end detection, simulation, junction-finding) are inlined into `agent.py` because the arena loader only adds the agent's own folder to `sys.path`.

## Model Architecture (`model.py`)

**PacmanCNN** — small 2-layer CNN-DQN:

```
Input: map_state [B, 1, 21, 21] + last_move one-hot [B, 4]

CNN Feature Extractor:
  Conv2d(1→32, 3×3, stride=1, pad=1) → ReLU  → [B, 32, 21, 21]
  Conv2d(32→64, 3×3, stride=2, pad=1) → ReLU  → [B, 64, 11, 11]
  Flatten                                        → [B, 7744]

DQN Head:
  Linear(7744 + 4 → 256) → ReLU
  Linear(256 → 4)        → Linear (raw Q-values)

Output: Q-values for [UP, DOWN, LEFT, RIGHT]
```

- ~2M parameters → ~7.6MB `.pt` file
- No BatchNorm (simpler inference, faster)
- Kaiming initialization for conv/hidden layers, Xavier for output layer
- Dropout (p=0.1) on the hidden linear layer during training only

**State encoding:**
- Wall cells (`1`) → `1.0`
- Empty cells (`0`) → `0.0`
- Pacman position → `2.0`
- Ghost position (if visible) → `3.0`
- Fog/unseen cells (`-1`) → `-1.0`

The CNN learns to treat fog cells differently from known empty cells.

## PacmanAgent Logic (`agent.py`)

### Decision Flow

```
step(map_state, my_position, enemy_position, step_number)
│
├─ Update internal map memory (merge fog observations over time)
│
├─ IF enemy_position is NOT None:
│   ├─ Run DQN forward pass → get Q-values for 4 moves
│   ├─ IF max(Q) - mean(Q) > confidence_threshold:
│   │   └─ Use DQN move (high confidence)
│   └─ ELSE:
│       └─ A* pathfind to enemy_position → use first move on path
│
├─ IF enemy_position is None:
│   ├─ IF last_known_enemy_pos exists and steps_since_seen <= 10:
│   │   └─ A* to last_known_enemy_pos
│   ├─ ELSE:
│   │   └─ A* to nearest frontier cell (explore unknown map areas)
│
└─ Compute speed steps (1 or 2) based on pacman_speed and path straightness
   Return (Move, steps)
```

### Key Design Choices

- **Confidence threshold on DQN**: `max(Q) - mean(Q)` must exceed a threshold. Prevents blindly following a DQN that is uncertain (common in fog-of-war states the DQN hasn't seen enough during training).
- **Map memory**: builds a persistent internal map by merging visible cells each step. A* can pathfind through previously-seen fog areas using this memory.
- **Frontier exploration**: when Ghost is lost for too long, find boundary cells between known and unknown map areas — smarter than random walk.
- **Speed-2 handling**: only use `steps=2` when the A* path is straight for 2+ tiles (no overshooting turns).
- **Timeout safety**: DQN is a single forward pass (~1ms). A* on 21×21 grid completes in <5ms. No iterative deepening. Total step time well under 100ms.

### A* Implementation

From submission 1's `PacmanAgent.astar_to_any_goal()`, adapted to work with the internal map memory. Uses Manhattan distance heuristic. Returns a list of `Move` enums representing the path.

## Training Script (`train.py`)

### Training Approach

Self-play against a rule-based Ghost opponent using the same logic from submission 1's GhostAgent (BFS flee, dead-end avoidance, simulation).

### Training Loop

- **Epochs**: configurable (default 100), each with N episodes (default 20)
- **Per episode**: reset environment, run up to 200 steps
- **Per step**: epsilon-greedy action selection → execute move → store transition in replay buffer → sample batch → gradient update → soft-update target network

### DQN Components

| Component | Value |
|-----------|-------|
| Replay buffer capacity | 50,000 |
| Min replay size before training | 1,000 |
| Target network update | Soft (τ=0.005) every step |
| Double DQN | Yes (online selects, target evaluates) |
| Loss function | SmoothL1 (Huber) |
| Optimizer | Adam, lr=1e-3 |
| Epsilon schedule | 1.0 → 0.05, decay 0.995/epoch |
| Gradient clipping | max_norm=10.0 |
| Batch size | 64 |
| Discount factor (γ) | 0.99 |

### Reward Shaping

| Event | Reward |
|-------|--------|
| Catch ghost | +100.0 |
| Getting closer (per cell Manhattan delta) | +2.0 |
| Moving away (per cell Manhattan delta) | -2.0 |
| Wall bump (stayed in place) | -1.0 |
| Time penalty per step | -0.1 |
| Timeout (ghost survives) | -50.0 |

### Fog of War in Training

The training environment supports configurable `pacman_obs_radius`. When set, the agent only sees within that radius, and `enemy_position` becomes `None` when the Ghost is outside range. This trains the DQN on partial-observability states so it doesn't panic when it can't see the Ghost.

### Saving

- Best model (by catch rate) → `best_pacman_dqn.pt`
- Periodic checkpoints → `pacman_dqn.pt`
- Save every 10 epochs and whenever catch rate improves

### CLI Arguments

```
python train.py                              # Default: 100 epochs, large model
python train.py --epochs 200                 # More epochs
python train.py --epochs 500 --lr 0.0005     # Custom epochs + learning rate
python train.py --obs-radius 5               # Train with fog of war
```

## GhostAgent (`agent.py`)

The GhostAgent is copied from submission 1's `hide_agent/` modules, flattened into `agent.py` as inline helper functions.

### Inlined Functions

- `bfs()` — shortest path between two positions
- `get_neighbors()` — valid adjacent cells
- `translate_move()` — convert position delta to `Move` enum
- `build_dead_end_exit_map()` — precompute dead-end cells → their exit gates
- `find_safest_junction()` — find junction farthest from threat within range
- `simulate()` — simulate N turns of chase, return final distance
- `enemy_next_position()` — predict where Pacman moves next

### GhostAgent Logic (unchanged from submission 1)

1. First step: build dead-end exit map
2. If enemy close (< 5 BFS steps): flee to farthest reachable cell
3. If in dead-end and can reach exit before Pacman: follow forced exit path
4. Otherwise: simulate moves, pick candidate with highest survival score
5. Anti-fidgeting: detect oscillation patterns, redirect to junctions

No DQN for Ghost — the rule-based logic is already strong and fast.
